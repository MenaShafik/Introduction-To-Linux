Identifying & Inspecting Filesystems

$ lsblk # block device tree — quick overview
$ lsblk -f # show filesystem type + UUID + label NAME FSTYPE LABEL UUID MOUNTPOINT sda1 xfs a1b2-... /boot sda2 xfs c3d4-... /
$ blkid # show UUID and type of all block devices /dev/sda1: UUID="a1b2-..." TYPE="xfs" /dev/sda2: UUID="c3d4-..." TYPE="xfs"
$ blkid /dev/sda1 # UUID of specific partition
$ df -h # disk space usage (human readable) Filesystem Size Used Avail Use% Mounted on /dev/sda2 19G 5G 14G 27% / /dev/sda1 1.0G 200M 800M 20% /boot tmpfs 3.8G 0 3.8G 0% /dev/shm
$ df -hT # disk space + filesystem TYPE
$ du -sh /var/log/ # disk usage of a directory 245M /var/log/
$ du -sh /var/log/* # size of each item in directory

mount — Manual Mounting

$ mount /dev/sdb1 /mnt/data # mount partition to /mnt/data
$ mount -t xfs /dev/sdb1 /mnt/data # specify filesystem type
$ mount -t iso9660 /dev/sr0 /mnt/cdrom # mount CD/DVD
$ mount -o ro /dev/sdb1 /mnt/data # mount READ-ONLY
$ mount -o remount,rw /mnt/data # remount as read-write
$ mount # show ALL currently mounted filesystems
$ mount | grep sdb # check if specific device is mounted
Manual mounts are temporary — lost after reboot! Add to /etc/fstab for permanent mounting.
⏏️

umount — Unmounting

$ umount /mnt/data # unmount by mount point
$ umount /dev/sdb1 # unmount by device name
$ umount -l /mnt/data # lazy unmount — detaches when not busy
$ umount -f /mnt/data # force unmount (NFS/network filesystems)
Cannot unmount busy filesystem! If you get "target is busy" — find what's using it:
$ lsof /mnt/data # list open files on that mount
$ fuser -m /mnt/data # show PIDs using the mount point

/etc/fstab — Permanent Mounts

/etc/fstab — 6 fields per line # Device           Mount   Type  Options  Dump Pass
UUID=a1b2-c3d4   /boot   xfs   defaults  0   0
UUID=e5f6-g7h8   /       xfs   defaults  0   0
/dev/sdb1        /data   xfs   defaults  0   0
/dev/sdc1        /backup ext4  ro,noexec 0   0
/dev/sr0         /mnt/cd iso9660 ro,noauto 0   0
fstab field explanations
1DeviceUUID=... or /dev/sda1 or LABEL=name
2Mount PointDirectory to mount on (/ /boot /data)
3FS Typexfs · ext4 · vfat · swap · iso9660
4Optionsdefaults · ro · noexec · noauto · _netdev
5Dump0=no backup, 1=backup (legacy, use 0)
6Passfsck order: 0=skip, 1=root, 2=others
Mount OptionMeaning
defaultsrw,suid,dev,exec,auto,nouser,async
roRead-only mount
rwRead-write mount
noexecCannot run executables from this mount
noautoDon't mount at boot automatically
_netdevWait for network before mounting (NFS)
nofailDon't fail boot if device missing
userAllow regular users to mount
$ mount -a # mount ALL entries in /etc/fstab # use to TEST fstab without rebooting
$ systemctl daemon-reload # reload after editing /etc/fstab
Bad /etc/fstab = system won't boot! Always test with mount -a after editing. Use UUID instead of /dev/sda1 — device names can change!

Hard Links & Soft (Symbolic) Links

Hard Link
file.txt
↓ same inode
hardlink.txt
both point to same data block
$ ln file.txt hardlink.txt # create hard link
$ ls -li file.txt hardlink.txt # same inode number! 123456 -rw-r--r-- 2 ali ali file.txt 123456 -rw-r--r-- 2 ali ali hardlink.txt
Hard Link Rules
✔Works on same filesystem only
✔Data survives if original deleted
✘Cannot link to directories
✘Cannot cross filesystems
Soft (Symbolic) Link
softlink.txt
↓ points to path
file.txt
like a Windows shortcut
$ ln -s file.txt softlink.txt # create symbolic link
$ ln -s /etc/httpd/conf /home/ali/httpd-conf # symlink to a directory
$ ls -l softlink.txt lrwxrwxrwx. 1 ali ali 8 Mar softlink.txt -> file.txt
Soft Link Rules
✔Can link across filesystems
✔Can link to directories
✘Breaks if original is deleted (dangling)
✘Different inode from target

Finding Files

$ find /etc -name "*.conf" # find by name pattern
$ find /home -user ali # find files owned by ali
$ find /var -size +100M # find files larger than 100MB
$ find / -perm /4000 # find SUID files
$ find /tmp -mtime -7 # modified in last 7 days
$ find /home -type l # find symbolic links only

Exam Tips

⚠ FSTABA mistake in /etc/fstab can prevent boot! Always test with mount -a after editing. Use UUID not device names — they can change after reboot.
💡 UUIDGet UUID with blkid or lsblk -f. Copy-paste it into fstab — don't type it by hand!
✔ MOUNT -aAfter editing fstab, run mount -a to test. If it returns an error, fix fstab immediately before rebooting.
🔗 LINKSHard links: same inode, same filesystem, no directories. Soft links: different inode, cross filesystem, can link dirs. ls -l shows l for symlinks.
⚠ BUSYCan't umount? You're probably inside the directory — cd / first, then try umount again!
💡 df vs dudf = free space on filesystem. du = how much space a directory/file uses. Different tools for different questions!
✔ nofailAdd nofail option in fstab for non-critical mounts (USB, extra disks) — system boots even if device is missing.
🔍 findfind / -user ali finds all files owned by ali — useful after deleting a user to locate orphaned files.
