dnf — Install & Remove

# INSTALL $ dnf install httpd # install package + all dependencies
$ dnf install httpd mariadb php # install multiple packages at once
$ dnf install /path/to/package.rpm # install local RPM file via dnf
# REMOVE $ dnf remove httpd # remove package
$ dnf autoremove # remove unused dependency packages
dnf resolves dependencies automatically — it installs everything the package needs. rpm alone does NOT resolve dependencies.
🔄

dnf — Update Packages

$ dnf update # update ALL installed packages
$ dnf update httpd # update ONE specific package
$ dnf check-update # list available updates (don't install)
$ dnf upgrade # same as update (modern alias)
$ dnf downgrade httpd # roll back to previous version
$ dnf update --security # install security updates ONLY
🔍

dnf — Search & Info

$ dnf search httpd # search by name and description
$ dnf info httpd # show full info: version, size, repo, description Name : httpd Version : 2.4.57 Release : 8.el9 Repo : appstream Summary : Apache HTTP Server
$ dnf list installed # list ALL installed packages
$ dnf list available # list ALL available packages in repos
$ dnf provides /var/www/html # which package provides this file/path? httpd-2.4.57 : Apache HTTP Server
$ dnf provides ifconfig # find which package provides a command
🎯

dnf — Groups

$ dnf group list # list all available package groups Available Environment Groups: Server with GUI Server Minimal Install Available Groups: Development Tools System Tools
$ dnf group info "Development Tools" # what packages are in this group?
$ dnf group install "Development Tools" # install all packages in group
$ dnf group remove "Development Tools" # remove all packages in group
Groups install many related packages at once — e.g. "Development Tools" installs gcc, make, git, etc. in one command.
📜

dnf — History & Undo

$ dnf history # list all past dnf transactions ID | Command | Date | Action 5 | install httpd | 2026-03-07 | Install 4 | update | 2026-03-06 | Update 3 | remove nginx | 2026-03-05 | Remove
$ dnf history info 5 # details of transaction #5
$ dnf history undo 5 # UNDO transaction #5 (uninstall httpd)
$ dnf history redo 5 # REDO transaction #5
dnf history undo is a lifesaver — if an update broke something, undo it with one command!

rpm — Low-level Tool

$ rpm -qa # list ALL installed packages httpd-2.4.57-8.el9.x86_64 bash-5.1.8-6.el9.x86_64 ...
$ rpm -q httpd # is httpd installed? shows version httpd-2.4.57-8.el9.x86_64
$ rpm -qi httpd # detailed info about installed package
$ rpm -ql httpd # list ALL files installed by httpd
$ rpm -qf /etc/httpd/conf/httpd.conf # which package owns this file? httpd-2.4.57-8.el9.x86_64
$ rpm -qd httpd # list documentation files for package
Never use rpm -i to install on a live system — it doesn't resolve dependencies. Use dnf install instead!
🌊

dnf Modules & Streams (AppStream)

$ dnf module list # list all available modules + streams Name Stream Profiles Summary php 8.1 common PHP scripting php 8.2 common PHP scripting nodejs 18 common Node.js mariadb 10.5 server MariaDB DB
$ dnf module list php # list streams for php module only
$ dnf module info php:8.2 # details about php stream 8.2
$ dnf module enable php:8.2 # enable specific version stream
$ dnf module install php:8.2 # enable + install php 8.2
$ dnf module disable php # disable php module entirely
$ dnf module reset php # reset module to default state
Streams let you choose WHICH version of a package to install — e.g. PHP 8.1 vs 8.2. Only ONE stream per module can be active at a time.

Exam Tips

⚠ RPM vs DNFrpm does NOT resolve dependencies — always use dnf install for installing. Use rpm only for querying installed packages.
🔍 PROVIDESdnf provides is key for the exam — if asked "install the package that provides command X", use this first!
✔ HISTORYIf you accidentally install or remove something, dnf history undo [ID] reverses the entire transaction cleanly.
📁 REPO FILECustom repos go in /etc/yum.repos.d/ with a .repo extension. Set enabled=1 and gpgcheck=0 for lab repos without a GPG key.
⚠ MODULESOnly one stream per module can be active at a time. To switch PHP from 8.1 to 8.2: dnf module reset php first, then enable 8.2.
💡 -qfrpm -qf /path/to/file — use this to find which package a config file belongs to. Very useful for troubleshooting!
✔ GROUPGroups are quoted when they contain spaces: dnf group install "Development Tools" — the quotes are required!
🔄 UPDATEdnf update with no arguments updates EVERYTHING. On exam, be careful — only update what the question asks for.
