Unit File Locations

PathPurpose
/usr/lib/systemd/system/Vendor / package-installed unit files (do NOT edit)
/etc/systemd/system/Admin customizations — overrides vendor files
/run/systemd/system/Runtime units (temporary, lost on reboot)
$ cat /usr/lib/systemd/system/sshd.service # view the sshd unit file
$ systemctl cat sshd # safer way — shows effective unit file
Rule: Never edit files in /usr/lib/systemd/system/. Put overrides in /etc/systemd/system/ instead.
🎮

systemctl — Service Control

$ systemctl start sshd # start service RIGHT NOW
$ systemctl stop sshd # stop service RIGHT NOW
$ systemctl restart sshd # stop then start (brief downtime)
$ systemctl reload sshd # reload config WITHOUT stopping
$ systemctl reload-or-restart sshd # reload if supported, else restart
$ systemctl enable sshd # start at BOOT (persistent) Created symlink /etc/systemd/system/...
$ systemctl disable sshd # do NOT start at boot
$ systemctl enable --now sshd # enable AND start immediately ← use this!
$ systemctl disable --now sshd # disable AND stop immediately
$ systemctl mask sshd # COMPLETELY prevent starting (even manually)
enable vs start: start = runs NOW (lost on reboot). enable = runs at BOOT (not now). Use enable --now to do both at once!
🚫

mask & unmask

$ systemctl mask sshd # links unit to /dev/null — impossible to start Created symlink /etc/systemd/system/sshd.service → /dev/null
$ systemctl start sshd # try to start masked service Failed to start sshd.service: Unit is masked.
$ systemctl unmask sshd # remove mask — can start again
ActionCan start manually?Starts at boot?
enabled✔ Yes✔ Yes
disabled✔ Yes✘ No
masked✘ No✘ No

systemctl status & Inspecting Services

$ systemctl status sshd ● sshd.service - OpenSSH server daemon Loaded: loaded (/usr/lib/systemd/system/sshd.service; enabled; vendor preset: enabled) Active: active (running) since Mon 2026-03-07 10:00:01 UTC; 2h ago Main PID: 1234 (sshd) CGroup: /system.slice/sshd.service └─1234 /usr/sbin/sshd -D Mar 07 10:00:01 host sshd[1234]: Server listening on 0.0.0.0 port 22
Reading the status output
● green active/running
● red failed
● white inactive/dead
Loaded: unit file found + boot setting (enabled/disabled)
Active: current runtime state + since when
Main PID: process ID of the service
CGroup: control group — all child processes
$ systemctl is-active sshd active
$ systemctl is-enabled sshd enabled
$ systemctl is-failed sshd failed ← or active
$ systemctl list-dependencies sshd # what does sshd depend on?
📜

Listing Units

$ systemctl list-units # all ACTIVE units
$ systemctl list-units --all # active + inactive + failed
$ systemctl list-units --type=service # only .service units
$ systemctl list-units --state=failed # show only FAILED units
$ systemctl list-unit-files # all unit files + enabled/disabled state
$ systemctl list-unit-files --type=service # service unit files only

Targets — Replacing Runlevels

Old Runlevelsystemd TargetDescription
0poweroff.targetShutdown
1rescue.targetSingle user / rescue
2,3,4multi-user.targetMulti-user, no GUI
5graphical.targetMulti-user + GUI
6reboot.targetReboot
$ systemctl get-default # show current default target graphical.target
$ systemctl set-default multi-user.target # set default boot target (no GUI)
$ systemctl isolate rescue.target # switch target NOW (like changing runlevel)
$ systemctl list-units --type=target # list all active targets
get-default = what boots next time. isolate = switch immediately without rebooting. These are different!
📰

journalctl — Service Logs

$ journalctl -u sshd # all logs for sshd service
$ journalctl -u sshd -f # FOLLOW live logs (like tail -f)
$ journalctl -u sshd -n 20 # last 20 lines only
$ journalctl -u sshd --since "1 hour ago" # logs from last hour
$ journalctl -xe # latest logs + explanations (debug failures)
Tip: When a service fails to start, always run journalctl -xe or journalctl -u servicename to find the error.

Exam Tips

⚠ TRAPenable does NOT start the service now! Use enable --now to both enable and start in one command.
🎯 TARGETset-default changes next boot. isolate changes NOW. Know the difference for the exam!
✔ DEBUGService fails? Run systemctl status sshd then journalctl -xe to find exact error.
📖 FILESNever edit /usr/lib/systemd/system/ — put changes in /etc/systemd/system/ to survive package updates.
⚠ MASKmask is stronger than disable — masked services cannot be started manually OR at boot.
💡 RELOADUse reload instead of restart when possible — no service interruption for active connections.
✔ VERIFYAfter enabling: check with systemctl is-enabled AND systemctl is-active separately.
🔍 LISTUse systemctl list-units --state=failed to quickly find any broken services on the system.
