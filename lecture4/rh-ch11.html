Logging Architecture

Applications / Kernel / Services
↓
systemd-journald
→
rsyslog
↓
/var/log/* (persistent text files)
ComponentRole
systemd-journaldCollects ALL log messages, stores in binary journal
rsyslogReads from journal, writes to /var/log/ text files
journalctlTool to query the systemd journal
logrotateRotates, compresses, archives old log files
Key: journald = fast binary storage. rsyslog = persistent text files. Both run together on RHEL.

journalctl — Query the Journal

Basic
$ journalctl # ALL journal entries (oldest first)
$ journalctl -r # newest entries FIRST
$ journalctl -f # FOLLOW — live stream of new logs
$ journalctl -n 20 # show last 20 lines
$ journalctl -e # jump to END of journal
Filter by Service / Priority
$ journalctl -u sshd # logs for sshd service only
$ journalctl -u sshd -f # follow sshd logs live
$ journalctl -p err # only errors (priority 3+)
$ journalctl -p warning..err # warning to error range
$ journalctl -xe # end of journal + explanations (debug failures)
Filter by Time
$ journalctl --since "today" # logs from today only
$ journalctl --since "1 hour ago" # last hour of logs
$ journalctl --since "2026-03-07 09:00" --until "2026-03-07 10:00" # specific time window
$ journalctl -b # logs since last BOOT only
$ journalctl -b -1 # logs from PREVIOUS boot
$ journalctl _UID=1001 # logs from user with UID 1001
$ journalctl -k # kernel messages only (like dmesg)
$ journalctl --list-boots # list all recorded boot sessions
💾

Journal Persistence

Default: Journal is stored in /run/log/journal/ — lost on reboot! To keep logs permanently, configure persistence.
$ mkdir -p /var/log/journal # create dir → journal becomes persistent automatically
$ vim /etc/systemd/journald.conf # edit journal config [Journal] Storage=persistent # auto / volatile / persistent / none SystemMaxUse=500M # max disk space for journal SystemKeepFree=100M # keep this much free MaxRetentionSec=1month
$ systemctl restart systemd-journald # apply changes
Storage=Where logs go
volatile/run/log/journal/ (RAM, lost on reboot)
persistent/var/log/journal/ (disk, survives reboot)
autopersistent if /var/log/journal/ exists, else volatile
noneDiscard all logs

logrotate

/etc/logrotate.conf
Main config — global defaults
/etc/logrotate.d/
Per-application rotate configs (syslog, httpd...)
/var/log/messages { weekly # rotate every week rotate 4 # keep 4 old copies compress # gzip old logs missingok # no error if file missing notifempty # skip if empty sharedscripts postrotate /bin/kill -HUP rsyslogd endscript }
$ logrotate -f /etc/logrotate.conf # force rotation NOW (test it)

logger — Write to syslog

$ logger "Hello from the CLI" # writes to /var/log/messages
$ logger -p auth.warning "Failed login attempt" # specify facility.priority
$ logger -p local0.info "Backup completed" # use local facility
$ tail -5 /var/log/messages # verify message was written Mar 07 10:00:01 host ali: Hello from the CLI
Use logger in shell scripts to write status messages directly into the system log — great for tracking automated tasks.

Exam Tips

⚠ PERSISTJournal is NOT persistent by default! Create /var/log/journal/ or set Storage=persistent in journald.conf.
🔍 DEBUGService fails? Run journalctl -xe immediately — it shows the error with context and explanation.
✔ FILES/var/log/secure = who logged in. /var/log/messages = general system. Know which file to check first.
⚠ RSYSLOGAfter editing /etc/rsyslog.conf always run systemctl restart rsyslog — changes won't apply otherwise.
📅 TIMEjournalctl -b = this boot only. journalctl -b -1 = previous boot. Useful when system crashed and rebooted.
✔ LOGGERUse logger to TEST if rsyslog rules work — write a test message and check if it lands in the right file.
